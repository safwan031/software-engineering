2. Business Logic coordinate operations between different modules by being implemented using server-side programming language.

3. Data Access Logic act as a method for users to retrieve and store data with the database layer.

4. Patient information, appointments, medical records, billing details, and inventory records.

5. RDBMS offers more structured data organization,data integrity and consistency.

6. When registering a new patient, the details input by the medical staff is received by business logic component. After validation, data access logic is instructed to store the patient detail in database.

7. User interface displays available slots, upon selection, sends appointment details to backend. Business logic validates and instructs data access logic to update appointment schedule.

8. Business Logic component coordinates with data access logic to store and retrieve medical record from database.
